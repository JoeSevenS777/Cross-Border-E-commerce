# 采购单 & Bigseller采购导入

## 功能概述

该宏用于从以下工作表导出"Place Order"数据：

-   萌睫
-   采菁
-   Flortte
-   夹子

并生成两类文件： - Joe77采购单\_`<店铺>`{=html}yyyymmdd.xlsx -
Bigseller采购导入\_`<店铺>`{=html}yyyymmdd.xlsx

------------------------------------------------------------------------

## 关键更新（2026版）

### 1. Action识别逻辑（已增强）

支持以下所有形式： - Place Order - Place Order - Network - Place Order -
Default

规则：**只要包含"place order"即视为有效**

------------------------------------------------------------------------

### 2. 采菁输出逻辑（已统一）

  项目               旧逻辑        新逻辑
  ------------------ ------------- ---------------------------
  Joe77采购单_采菁   独立8列结构   ✅ 与萌睫完全一致
  输出字段           多列          商品链接 / 属性SKU / 数量

------------------------------------------------------------------------

### 3. 文件保存路径（已统一）

所有店铺统一为：

    ThisWorkbook.Path

即：**与宏文件同目录**

  店铺      Joe77路径             Bigseller路径
  --------- --------------------- ---------------
  萌睫      同目录                同目录
  采菁      ✅ 同目录（已修改）   同目录
  Flortte   同目录                同目录
  夹子      同目录                同目录

------------------------------------------------------------------------

## 各店铺规则

### 萌睫 / 采菁

Joe77字段： - 商品链接 - 属性SKU - 数量

------------------------------------------------------------------------

### Flortte / 夹子

Joe77字段： - SKU - 数量

并自动去除前缀： - MJ-Flortte- - MJ-夹子- - MJ-JiaZi-

------------------------------------------------------------------------

## Bigseller导入格式

固定18列结构（含表头）： - SKU Name - Purchase Qty - 及其他系统字段

------------------------------------------------------------------------

## 文件命名规则

    Joe77采购单_<店铺>yyyymmdd.xlsx
    Bigseller采购导入_<店铺>yyyymmdd.xlsx

------------------------------------------------------------------------

## 自动行为

-   若当天文件不存在：
    -   自动重命名旧文件
-   若已存在：
    -   覆盖数据（保留结构）

------------------------------------------------------------------------

## 使用方式

1.  打开对应工作表（萌睫 / 采菁 / Flortte / 夹子）

2.  运行宏：

        Joe采购单_Bigseller采购导入

3.  自动生成文件

4.  可选择自动打开结果

------------------------------------------------------------------------

## 注意事项

-   必须包含字段：
    -   action
    -   SKU / 属性SKU / 商品链接 / 数量
-   工作簿必须已保存
-   仅对指定4个工作表有效

------------------------------------------------------------------------

## 设计原则

-   统一结构（降低维护成本）
-   宽松匹配（适应业务变化）
-   无外部路径依赖（提升稳定性）
